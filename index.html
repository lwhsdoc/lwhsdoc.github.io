3.vim基本使用方法

vim简单入门

2024年3月2日

3:09

虽然说大部分发行版默认编辑器不是vim而是nano，但是他的通用性很好，在Linux和FreeBSD等等系统上都可以用。

这里介绍他最基本的使用方法↓

首先，你要确定你的系统有没有安装vim

vim -v

如果有反应，并且他打开了一个显示版本号的页面，你可以输“：q”退出，如果报错提示“command not found”就请安装它

Debian/Ubuntu↓

sudo apt install vim -y

rhel/alma/rocky因为自带vim所以说不需要单独安装

使用方法↓

vim默认情况下vim是命令模式，按下i进入编辑模式以后你就可以像普通的文本编辑器编辑文本。

保存、退出或者是保存后退出或者是强制退出需要进入命令模式，从编辑模式退出到命令模式按下esc退出。

命令模式下输入退出vim::q（如果你对文本做了更改他会报错）

保存更改输入:w

保存更改并退出输入:wq

强制退出不保留更改输入:q!

请注意，仅仅适用于最基本的使用，仅仅适用于日常编辑文本的工作。提醒一句，如果你要查看重要文本，而不想对他做出任何的整改，防患于未然，请用view。

2.git入门

Git是一个分布式版本控制系统，它允许开发者在本地工作并保存完整的项目历史。

Git的基本概念↓

**远程仓库（Remote）：通常托管在像GitHub这样的代码托管平台上，用于与他人共享代码和协作。 **

**工作区（Workspace）：这是git当前正在工作的目录，它包含了文件和文件夹。 **

**暂存区（Index/Stage）：当更改了一些文件并想要提交这些更改时，首先需要将这些更改添加到暂存区。 **

版本库（Repository）：这是Git的核心部分，它存储了项目的所有历史记录和元数据。

**安装Git↓

可以访问Git的官方网站下载并安装适合您操作系统的Git版本。**

或者是通过包管理器安装↓

Debian↓

sudo apt install git -y

rhel/alma/rocky↓

sudo dnf in git -y

版本控制↓

创建版本库↓

使用git init命令在现有项目中初始化一个新的Git仓库，或者使用git clone url.git复制一个远程仓库到本地。

版本回退↓

可以使用git checkout命令切换到特定的提交，进行版本回退。

管理修改↓

使用git add将更改添加到暂存区，然后使用git commit将暂存区的更改提交到仓库中。

提交流程↓

提交文件↓

文件进行更改后，通过git add和git commit进行本地提交。

推送到远程仓库↓

使用git push将本地的提交推送到远程仓库以同步更改。

撤销操作↓

如果不小心提交了错误或想要撤销某些更改，可以使用git reset或git revert等命令进行撤销。

分支↓

创建分支↓

**使用git branch命令来创建新分支。

切换分支↓**

使用git checkout命令在不同分支之间切换。

合并分支↓

使用git merge命令将一个分支的更改合并到另一个分支。
